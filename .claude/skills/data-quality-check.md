# 報告品質檢查框架（兩層制）

檢查分為 🔴 硬門檻（未通過則阻斷發送）和 🟡 軟門檻（警告但放行）。
Hook（validate-report.py）自動執行第二道防線。

## 1. 股票代碼校驗

### 高風險混淆對照表
| 容易搞混的 | 正確代碼 | 錯誤代碼 | 備注 |
|-----------|---------|---------|------|
| 群聯 | 8299 | 8046 | 8046 是南電 |
| 南電 | 8046 | 8299 | 8299 是群聯 |
| 華通 | 2313 | - | PCB + 低軌衛星雙族群 |
| 技嘉 | 2376 | - | AI 伺服器 + NB/PC 雙族群 |
| 微星 | 2377 | - | AI 伺服器 + NB/PC 雙族群 |

### 校驗規則
- 每個提及的個股代碼必須與代碼表交叉比對
- 不在代碼表中的個股，僅在新聞明確標注代碼時才引用
- 同一個股在報告中出現多次時，代碼必須一致

## 2. 模糊數字偵測

### 禁止用語（關鍵數據位）
- `~` `約` `大約` `接近` `附近` `左右`
- `X` `XX` `YY.X` `31.X`（半精確數字）
- `平盤附近` `小漲` `小跌`（無具體數字）

### 允許用語
- 精確數字：`21,345.67 (+1.23%)`
- 明確省略：整項不寫
- 方向性描述（非數據位）：「偏多」「觀望」「謹慎」

### 檢查方法
掃描報告中所有數字位置，確認：
- 指數：必須精確到個位數
- 漲跌幅：必須精確到小數點後兩位
- 匯率：必須精確到小數點後兩位
- 法人金額：必須精確到億元
- 個股價格：必須精確到元

## 3. 空殼區段檢查

### 禁止出現的空殼模式
- 「尚未取得」「待補」「待確認」「資料截至蒐集時未取得」
- 只有標題但內容為空的區段
- 只有模板格式但未填入數據的區段

### 正確處理方式
- 取不到 → 整個區段省略（不留標題）
- 部分取得 → 列出已取得的，標注來源
- 導引其他報告 → 「詳見盤後深度報告」（僅盤前版可用）

## 4. 匯率來源檢查

### 禁止行為
- 用 ADR 價格 ÷ 台股價格反推匯率
- 用歷史匯率假設當日匯率
- 編造匯率數字

### 正確來源
- Google 搜尋結果中的即時匯率
- Investing.com 匯率頁面
- 鉅亨網外匯頁面
- 新聞報導中引用的央行或銀行匯率

## 5. Slack 格式檢查

### 必須使用
- 粗體：`*文字*`（單星號）
- 斜體：`_文字_`
- 列表：`•` 開頭

### 禁止使用
- `**雙星號**`（Slack 不支援）
- `# 標題`（Slack 不支援）
- HTML 標籤（`<b>` `<br>` 等）
- Markdown 連結 `[文字](URL)`（應用 `<URL|文字>`）

## 6. 內容合規檢查

### 禁止用語
- 「建議買進」「建議賣出」「強烈推薦」
- 「保證獲利」「必漲」「必跌」

### 允許用語
- 「值得關注」「需留意」「可觀察」
- 「估值偏高/合理」「短線壓力位」「支撐有效」

## 7. 搜尋次數確認

| 報告類型 | 搜尋上限 | 時間上限 |
|---------|---------|---------|
| 台股盤前 | 7 次 | 15 分鐘 |
| 台股盤後 | 8 次 | 20 分鐘 |
| 美股盤前 | 9 次 | 20 分鐘 |

## 8. 訊息長度確認

每則 Slack 訊息不得超過 3,500 字。若接近上限：
- 精簡描述，非刪除區段
- 優先保留核心分析（量價判讀、傳導鏈、條件預判）
- 次要資訊（格式裝飾、過長的背景說明）優先刪減
