# Ground Truth 驗證框架（台股版）

## GT 登記表

第 0.5 步 WebFetch 取得的數據為 ground truth。撰寫每則訊息前，
必須建立內部「GT 登記表」並逐項比對：

- GT_INDEX：加權指數收盤 = [第 0.5 步數字]
- GT_CHANGE：漲跌幅 = [第 0.5 步數字]
- GT_FX：USD/TWD = [第 0.5 步數字]
- GT_DATE：數據日期 = [第 0.5 步日期]

## 🔴 硬門檻（任一不過 → 該則訊息禁止發送，改發品質警告）

1. 指數與 GT_INDEX 差距 > 50 點 → BLOCK
2. 漲跌幅方向與 GT_CHANGE 正負相反 → BLOCK
3. 股票代碼不在代碼表中且無新聞來源確認 → BLOCK
4. 關鍵數據位出現模糊數字（~、約、X）→ BLOCK
5. 「即將舉行」事件未經時效校驗即引用 → BLOCK

## 🟡 軟門檻（記錄但繼續發送）

6. 搜尋次數達上限
7. 部分區段省略
8. 事件時效無法確認（標註「待確認」即可）

## 🔴 硬門檻觸發時的替代訊息

```
⚠️ *品質檢查未通過*
本則報告因以下問題暫停發送：• [具體問題] • [GT 數據 vs 報告數據]
_關鍵數據請直接查閱 cnyes.com_
```

發送替代訊息後，嘗試修正問題並重新撰寫。修正後仍不過則保留替代訊息，繼續下一則。
