# 發送前品質門檻（兩層防線）— 台股版

## 第一道防線：GT 自我檢查（撰寫完成後、發送前）

### 🔴 硬門檻（任一未過 → 修正後才能發送）
1. □ 指數/匯率數字與 GT 登記表一致（差距 ≤ 50 點）
2. □ 所有股票代碼已與代碼表核對（群聯=8299, 南電=8046）
3. □ 無模糊數字於關鍵數據位（指數、漲跌幅、匯率）
4. □ 未引用未經時效校驗的「即將舉行」事件
5. □ 匯率非自行計算（ADR 反推）

### 🟡 軟門檻（記錄但不阻斷）
6. □ 搜尋次數未超限
7. □ 無空殼區段
8. □ Slack mrkdwn 格式正確
9. □ 每則訊息 ≤ 3500 字
10. □ 無投資建議用語

## 第二道防線：Hook 自動攔截（validate-report.py）

硬門檻不過 = 訊息被阻擋，需修正重發
軟門檻不過 = 收到警告但允許發送
